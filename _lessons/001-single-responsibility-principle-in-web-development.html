There are two principles know in the field of Software Engineering, both of whom are very beneficial in managing any kind of software system in a simple and powerful way.
These are:

  • Single Responsibility Principle
  • DRY: Don't repeat yourself

Single Responsibility Principle

The single responsibility principle states that every piece of a system (file, module or class) should have responsibility over a single part of the functionality provided by the software,
and that responsibility should be entirely encapsulated by the class.

Some rules:

  • Every single piece of the system should have a just one single purpose.
  • If several aspects apply to a certain area of a system, those different aspects should be implemented in seperate pieces: Separation of concerns

Web Development

When developing for the web there are three techniques to comprehend for the three main aspects of a web page:

  1. HTML to determine WHAT should be shown on the page: The content
  2. CSS to determine HOW it should be shown on the page: The layout
  3. JavaScript to add INTERACTION to the page: The layout
The first important this to learn is to keep all layout related information from the HTML.

HTML

Any HTML-file contains the content to show on screen, where all elements are enclosed in tags. Those tags adds a structure to the document and thereby inform any browser how to interpret the document.
Without any guidelines on HOW to render that document (CSS), all browsers have a default way on how to render a document and that will be largely dependent on which tags are found in the document. Some examples:

<p>
Paragraph of text
<ul>
Unordered List/dd>
<h1>, <h2>, ..., <h7>
Headers in various sizes
<division>
Division (of section)
<nav>
Navigation block

These are just a subset of a list of available tags.

After years of web development some best practices have emerged on when to use certain tags and how to apply styling to them: Bootstrap
When you start writing HTML, stick to the Fundamental HTML elements tags mentioned in the documentations of BootstrapsBase CSS
Note: That documentation is based upon the outdated version 2 of Bootstrap, but is still useful and valid for the HTML-tags in newer versions.
